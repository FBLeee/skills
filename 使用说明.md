根据 Matt Pocock 的 GitHub 仓库文档，这套 Skills 是专门为 **AI 编程助手（如 Claude Code 等命令行 AI Agent）** 设计的预设工作流和提示词指令。

要使用这套技能，你需要通过终端将它们安装到你的代码项目中，然后在与 AI 助手的对话中通过 `/指令` 来触发它们。以下是具体的使用步骤：

### 第一步：安装技能 (30秒快速设置)
在你的终端（Terminal）中，进入你的项目根目录，运行以下命令：
```bash
npx skills@latest add mattpocock/skills
```
运行后，**就会出现你刚才截图里的那个交互式菜单**。
1. 使用**空格键**勾选你想要的技能（建议初学者可以全选，或者根据需要选择）。
2. **非常重要**：确保你勾选了 `/setup-matt-pocock-skills` 这个技能。
3. 按回车确认，系统会问你要将这些技能安装到哪个 AI Agent 中（通常会写入到项目里的 `.claude` 目录或类似配置中）。

### 第二步：初始化配置
启动你的 AI 编程助手（例如在终端运行 `claude` 进入 Claude Code 交互界面）。
在 AI 的对话框中，输入并运行以下指令：
```text
/setup-matt-pocock-skills
```
AI 会开始引导你完成基础配置，它会问你三个问题：
1. 你想使用哪个 Issue 追踪器？（GitHub、Linear 还是本地文件）
2. 当你对工单进行分类时，你会使用什么标签？（用于 `/triage` 技能）
3. 你希望将 AI 创建的文档保存在哪个目录下？

回答完这些问题后，配置就完成了！

### 第三步：在日常开发中使用 (核心指令)
配置完成后，你就可以在与 AI 的日常对话中使用这些技能了。当你想让 AI 做特定任务时，直接输入对应的 `/指令` 即可。

以下是几个最常用、最核心的使用场景和指令：

**1. 开始写代码前（防止 AI 瞎写，对齐需求）**
*   输入 `/grill-me`：AI 会化身严厉的面试官，不断追问你的计划和设计细节，直到把所有模糊的边缘情况都理清。
*   输入 `/grill-with-docs`：和上面类似，但它还会帮你提取项目中的“行话（专业术语）”并更新到 `CONTEXT.md` 中，让 AI 以后更懂你的项目。

**2. 开始写代码时（保证代码质量）**
*   输入 `/tdd`：让 AI 使用“测试驱动开发（红-绿-重构）”的模式来写代码。AI 会先写一个失败的测试，然后再写代码让测试通过，这能极大地提高 AI 生成代码的可靠性。

**3. 遇到难搞的 Bug 时**
*   输入 `/diagnosing-bugs`：AI 会启动一套严谨的排查流程（复现 -> 最小化 -> 假设 -> 插桩 -> 修复 -> 回归测试），而不是像无头苍蝇一样乱改代码。

**4. 整理项目和文档时**
*   输入 `/to-prd`：AI 会把你们刚才讨论的所有内容，自动总结成一份标准的 PRD（产品需求文档）并发布。
*   输入 `/improve-codebase-architecture`：让 AI 扫描你现在的代码库，找出可以优化架构的地方，并生成一份可视化的 HTML 报告给你看。

**💡 终极提示**：如果你不知道现在该用哪个指令，可以直接在 AI 对话框输入 **`/ask-matt`**，告诉它你现在遇到了什么情况，它会像一个路由器一样，为你推荐最适合的技能指令。
